协议生成页面工作流

由于我们是金融公司,协议内容更新较为频繁,几乎每周都要维护协议,做一个协议全自动维护系统,技术难度大,开发用时时间长; 所以我考虑了一下,将工作流程简化,规范化;并开发出文本简化/文本对比等小工具,尽量使工作流程做到半自动化,提升了协议开发与维护效率

1.打开文档.删除所有批注/标记(很重要,不然生成文档会有多余字段)

2.另存为筛选过的页面

3.编辑器打开页面,选择正确编码格式

4.进入文件简化页面 https://jxj666.github.io/word_to_html/ 将word生成源代码实用化(word生成源码样式配置信息过多,结构冗余过多,很影响性能,并且过于繁杂不容易插入代码并且缺少生产环境配置信息)

5.讲代码复制粘贴到指定位置 微调样式,插值(插值方式尽量保证文件与后端保存的一致,web代码与php端代码保持高度一致便于维护,通过模板转化代码完成客户的环境赋值)

6.检查结果 打开对比页面: https://jxj666.github.io/diff_word/ 粘贴文档/页面文字,进行对比
